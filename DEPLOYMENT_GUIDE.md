### 步骤 3: 配置云开发环境

**操作步骤**:

1. **开通云开发**
   - 点击工具栏「云开发」按钮
   - 开通云开发环境 (免费版即可)
   - 记录环境 ID (如：cloud1-xxx)

2. **创建数据库集合**
   
   需要创建以下 6 个集合：
   
   | 集合名 | 说明 | 权限 |
   |--------|------|------|
   | `terms` | 术语库 | 所有用户可读 |
   | `users` | 用户数据 | 用户自己可读写 |
   | `daily_word` | 每日一词 | 所有用户可读 |
   | `categories` | 分类配置 | 所有用户可读 |
   | `badges` | 勋章配置 | 所有用户可读 |
   | `quiz_questions` | 测验题目 | 所有用户可读 |

3. **导入初始数据**
   
   **terms 集合**:
   - 导入文件：`data/terms_extended.json`
   - 数据量：100 个术语
   
   **categories 集合**:
   - 导入文件：`data/categories.json`
   - 数据量：7 个分类
   
   **badges 集合**:
   - 导入文件：`data/badges.json`
   - 数据量：8 个勋章
   
   **quiz_questions 集合**:
   - 导入文件：`data/quiz_questions.json`
   - 数据量：50+ 道题目

4. **修改云环境 ID**
   
   编辑 `app.js`，将云环境 ID 改为你的实际环境 ID：
   
   ```javascript
   wx.cloud.init({
     env: 'your-env-id'  // 替换为你的云环境 ID
   })
   ```

## 🐛 常见问题

### Q1: 云函数上传失败？

**解决方案**:
1. 检查是否已开通云开发
2. 检查 `package.json` 中依赖是否正确
3. 右键选择「云端安装依赖」后再上传

### Q2: 数据库导入失败？

**解决方案**:
1. 检查 JSON 文件格式是否正确
2. 确保集合已创建
3. 分批导入 (每次不超过 500 条)

### Q3: 真机预览空白？

**解决方案**:
1. 检查 AppID 是否正确
2. 检查云环境 ID 是否配置
3. 查看开发者工具控制台错误信息

### Q4: 星星动画不显示？

**解决方案**:
1. 检查 `pages/index/index.js` 中 `generateStars()` 函数
2. 确保 WXML 中有 `<view class="stars">` 元素
3. 检查 WXSS 中 `.star` 样式
