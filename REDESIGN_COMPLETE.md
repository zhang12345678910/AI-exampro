- **原方案**: 3 个功能模块
- **新方案**: 6 个功能模块（首页、词典、学习、我的、工具）

### 2. 术语库规模
- **原方案**: 50 词

### 3. 检索方式
- **原方案**: 关键词搜索

### 4. 学习体系
- **原方案**: 简单已学/未学标记
- **新方案**: 分级学习（入门/进阶/专业）+ 打卡 + 测试 + 勋章

### 5. 用户系统
- **原方案**: 基础微信登录
- **新方案**: 完整用户中心（个人信息、收藏夹、生词本、笔记、勋章墙）

### 6. 工具能力
- **原方案**: 基础分享
- **新方案**: 名词卡片生成（多模板）+ 多场景分享

### 1. 术语库扩展
- 需要大量时间整理 1000+AI 术语
- 建议：使用 AI 辅助生成 + 人工审核
- 优先级：入门 200 词 → 进阶 400 词 → 专业 400 词

### 2. UI 设计
- Tab 栏图标需要专业设计
- 建议：使用 Figma/Sketch 设计后导出
- 临时方案：使用 emoji + 文字

### 3. 云函数开发
- 需要新增 5 个云函数
- 建议：优先开发 P0 功能相关云函数
- 注意：云函数权限配置

### 4. 性能优化
- 术语库扩大后需要分页加载
- 图片需要懒加载
- 本地缓存需要优化
