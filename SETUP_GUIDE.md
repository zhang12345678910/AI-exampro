# AI 词星球小程序 - 配置指南

**版本**: 1.0  
**更新日期**: 2026-04-13

---

## 📋 配置清单

完成以下配置后，小程序才能正常运行。

---

## 1️⃣ 导入项目

### 步骤
1. 打开 **微信开发者工具**
2. 点击「+」或「导入项目」
3. 项目目录：`/home/admin/.openclaw/workspace/projects/ai-terms-miniprogram`
4. AppID：填入你的小程序 AppID（测试可选「测试号」）
5. 点击「导入」

### 注意事项
- 如果没有 AppID，可使用「测试号」进行开发
- 正式环境需要注册小程序账号

---

## 2️⃣ 开通云开发

### 步骤
1. 点击工具栏「云开发」按钮
2. 首次使用会提示开通，点击「开通」
3. 创建环境（建议命名：`ai-terms-env`）
4. 等待环境创建完成（约 1-2 分钟）

### 环境配置
- **环境名称**: ai-terms-env（自定义）
- **环境 ID**: 系统自动生成（如：cloud1-xxx）
- **计费模式**: 免费版（开发阶段够用）

---

## 3️⃣ 修改项目配置

### 修改 app.js
打开 `app.js`，将云环境 ID 替换为你的实际环境 ID：

```javascript
wx.cloud.init({
  env: 'ai-terms-env',  // ← 改为你的环境 ID（在云开发控制台查看）
  traceUser: true
})
```

### 修改 project.config.json
打开 `project.config.json`，填入你的 AppID：

```json
{
  "appid": "wxXXXXXXXXXXXXXXXX",  // ← 改为你的 AppID
  "projectname": "ai-terms-miniprogram"
}
```

---

## 4️⃣ 创建数据库集合

### 步骤
1. 在云开发控制台，点击「数据库」
2. 点击「+」创建集合
3. 创建以下 3 个集合：

| 集合名称 | 用途 | 权限 |
|---------|------|------|
| `terms` | 存储 AI 术语数据 | 所有用户可读 |
| `users` | 存储用户数据 | 仅用户自己可读写 |
| `quiz_questions` | 存储测验题目 | 所有用户可读 |

### 设置权限
- `terms`: 自定义规则 → 所有用户可读，仅创建者可写
- `users`: 自定义规则 → 仅用户自己可读写
- `quiz_questions`: 自定义规则 → 所有用户可读，仅创建者可写

---

## 5️⃣ 导入数据

### 导入术语数据
1. 在云开发控制台，选择 `terms` 集合
2. 点击「导入」→「文件导入」
3. 选择文件：`data/terms.json`
4. 点击「导入」

### 导入测验题目
1. 在云开发控制台，选择 `quiz_questions` 集合
2. 点击「导入」→「文件导入」
3. 选择文件：`data/quiz_questions.json`
4. 点击「导入」

---

## 6️⃣ 上传云函数

### 步骤
1. 在云开发控制台，点击「云函数」
2. 点击「+」→「本地导入」
3. 选择云函数目录，依次上传：

| 云函数 | 目录 | 用途 |
|--------|------|------|
| `login` | `cloud/login` | 用户登录 |
| `stats` | `cloud/stats` | 学习统计 |
| `toggleCollect` | `cloud/toggleCollect` | 收藏功能 |
| `saveQuizScore` | `cloud/saveQuizScore` | 保存成绩 |

### 上传后操作
- 每个云函数上传后，右键点击 →「依赖安装」
- 等待依赖安装完成（显示「✓」）

## 🔟 常见问题

### Q1: 云函数调用失败
**原因**: 云函数未上传或依赖未安装  
**解决**: 重新上传云函数，右键「依赖安装」

### Q2: 数据库读取失败
**原因**: 权限设置不正确  
**解决**: 检查集合权限设置，确保用户有读取权限

### Q3: 真机预览空白
**原因**: AppID 不匹配或云环境未配置  
**解决**: 检查 `project.config.json` 和 `app.js` 中的配置

### Q4: iOS 日期报错
**原因**: iOS 不支持横杠格式的日期  
**解决**: 代码中已处理，使用 `utils/date.js` 的 `parseDate` 函数
